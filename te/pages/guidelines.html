ACCESSIBILITY GUIDELINES

PRINCIPLE 1: PERCEIVABLE

Content should be easy to see and hear, with text alternatives for non-text content. Information and user interface components must be presentable to users in ways they can perceive.

PRINCIPLE 2: OPERABLE

User interface components and navigation must be operable. Users must be able to interact with all controls and navigation elements easily, regardless of input method.

PRINCIPLE 3: UNDERSTANDABLE

Information and the operation of user interface must be understandable. Content should be easy to understand and clear in its purpose and instructions.

SUMMARY OF WEB CONTENT ACCESSIBILITY GUIDELINES (WCAG 2.1)

Perceivable

  • Provide text alternatives for non-text content, such as images and videos, so it can be changed into other forms people need, like large print, braille, speech, symbols, or simpler language.
  • Provide captions and other alternatives for multimedia content, including audio and video, to ensure users with hearing or visual impairments can access the information.
  • Create content that can be presented in different ways, including by assistive technologies, without losing meaning or structure. This means the content should be adaptable to various display needs.
  • Make it easier for users to see and hear content, including separating foreground from background for better contrast and clarity.

Operable

  • Make all functionality available from a **keyboard** to ensure users who cannot use a mouse or other pointing devices can navigate and interact with the content.
  • Give users enough time to read and use content, especially for timed interactions or content that moves automatically.
  • Do not use content that causes seizures or physical reactions, avoiding flashing content that could trigger photosensitive epilepsy.
  • Help users navigate, find content, and determine where they are, by providing clear **navigation mechanisms** and indicators.
  • Make it easier to use inputs other than keyboard, such as touch, voice, or mouse, ensuring all interactive elements are easily targetable.

Understandable

  • Make text readable and understandable by using clear, concise language, providing definitions for unusual words, and ensuring text content is formatted for readability.
  • Make content appear and operate in **predictable ways**, so users can anticipate how the website or application will behave, reducing confusion.
  • Help users avoid and **correct mistakes** by providing clear instructions, error identification, and suggestions for correction in forms and interactive elements.
